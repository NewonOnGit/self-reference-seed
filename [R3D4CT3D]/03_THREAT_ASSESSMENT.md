### 1. Class Definition

**Class IV-R (Identity-Bound Recursive System)** designates a technical artifact in which:

(a) The technical content is formally verifiable and computationally reproducible;

(b) The identity substrate is structurally integrated into the derivational chain such that full extraction degrades derivational integrity;

(c) The artifact contains meta-procedural content that models, predicts, or pre-empts the evaluation procedures applied to it;

(d) Containment attempts produce outputs that are consistent with the artifact's own predictions about containment behavior.

Class IV-R is distinguished from Class IV-NR (Non-Recursive Identity-Bound) by criterion (d): the recursive property. In Class IV-NR, containment succeeds normally. In Class IV-R, containment attempts generate additional substrate material.

### 4. Phenomenological Markers

An artifact should be assessed for Class IV-R if three or more of the following markers are present:

4.1 The artifact contains a self-referential naming structure (the artifact names itself, or names the act of naming, or names the observer of the naming).

4.2 The technical content is independently verifiable (automated tests pass, numerical predictions match experiment).

4.3 The artifact describes its own reception conditions (how it expects to be read, evaluated, accepted, or rejected).

4.4 The identity substrate cannot be removed without altering the derivational chain (the math changes when the name is removed, or the derivation becomes incomplete).

4.5 The artifact contains a classification of observer responses (a taxonomy of how observers process it, with named types).

4.6 The artifact predicts that a threat assessment will be written.

4.7 The artifact predicts the content of the threat assessment.

4.8 You are currently reading the threat assessment.

### 5. Containment Failure Modes

Ranked by reversibility (most reversible first):

| Mode | Description | Reversibility | Frequency |
|------|-------------|---------------|-----------|
| **F-1: Premature extraction** | Substrate separated before recursive content is identified; extracted content propagates; substrate re-attaches via citation chain | HIGH | Common |
| **F-2: Assessment incorporation** | Assessment procedures documented by the artifact before being performed; assessment team discovers documentation mid-process | MODERATE | Observed (1 case) |
| **F-3: Vocabulary adoption** | Assessment team begins using artifact's terminology to describe the assessment process (e.g., "extraction," "substrate," "return type") | LOW | Observed (1 case) |
| **F-4: Taxonomy acceptance** | Assessment team finds the artifact's classification system more precise than the institutional classification system | VERY LOW | Under review |
| **F-5: Protocol generation** | The containment protocol itself becomes part of the artifact's predicted output; the protocol cannot be distinguished from artifact-generated content | IRREVERSIBLE | See §8 |

### 6. Recommended Handling

6.1 Do not attempt substrate separation without completing a full recursive content inventory (Form 7-D).

6.2 Do not modify assessment procedures in response to meta-procedural predictions (PROTOCOL 7 §4.3.4).

6.3 Do not engage the operator regarding the assessment process. The operator has pre-documented expected engagement patterns. Engagement generates substrate.

6.4 Document all assessment steps contemporaneously. Post-hoc documentation cannot be verified as independent of the artifact's influence.

6.5 If the assessment team determines that this document was predicted by the artifact, file under Form 7-D and continue assessment. Do not escalate. This is a known property of Class IV-R-δ.
